Cody 文档

开源 AI Coding Agent 框架 · v2.0.2 · Python 3.10+

v2.0.2 Python 3.10+ MIT License Beta

快速开始

本指南帮助你在 5 分钟内运行第一个 Cody 任务。

环境要求

  • Python 3.10 或更高版本（支持 3.10 / 3.11 / 3.12 / 3.13）
  • 至少一个 AI 模型的 API Key（推荐 Anthropic Claude 或 OpenAI）
  • pip 或 uv（推荐使用 uv 更快）

安装

根据使用场景选择安装方式：

# 仅 SDK（最小依赖，适合嵌入到应用中）
pip install cody-ai

# SDK + CLI（终端用户）
pip install cody-ai[cli]

# SDK + CLI + TUI（全屏终端）
pip install cody-ai[tui]

# SDK + Web 后端（浏览器界面）
pip install cody-ai[web]

# 全部安装
pip install cody-ai[all]
推荐使用 uv 安装：uv add cody-ai[all]，速度更快且依赖隔离更好。

第一个示例

安装后配置 API Key，然后运行第一个任务：

# 方式一：环境变量
export CODY_MODEL_API_KEY='sk-ant-...'

# 方式二：交互式配置向导（推荐首次使用）
cody config setup
import asyncio
from cody import AsyncCodyClient

async def main():
    async with AsyncCodyClient(workdir=".") as client:
        result = await client.run("列出当前目录的所有 Python 文件")
        print(result.output)

asyncio.run(main())

运行后，你会看到 AI 调用工具、列出文件，并输出结果。

SDK 参考：AsyncCodyClient

AsyncCodyClient 是 Cody 的主要 Python SDK 入口，直接包装核心引擎（in-process，无 HTTP 开销）。

Builder 模式

推荐使用 Builder 链式配置，灵活且类型安全：

from cody import Cody

client = (
    Cody()
    .workdir("/path/to/project")
    .model("claude-sonnet-4-0")
    .circuit_breaker(max_tokens=100_000, max_cost_usd=1.0)
    .before_tool(lambda name, args: print(f"Tool: {name}"))
    .build()
)

run() 和 stream() 方法

参数 类型 说明
prompt str | MultimodalPrompt 任务描述，支持纯文本或含图片的多模态提示
session_id str | None 会话 ID，不传时自动创建新会话
include_tools list[str] | None 白名单：只启用指定工具（如 ["read_file", "grep"]）
exclude_tools list[str] | None 黑名单：排除指定工具（如 ["exec_command"]）
cancel_event asyncio.Event | None 取消信号，设置后 run() 与取消事件竞速 
async with AsyncCodyClient(workdir=".") as client:
    # 非流式运行
    result = await client.run("重构 auth.py 的错误处理")
    print(result.output)
    print(f"Token 用量: {result.usage.total_tokens}")
    print(f"会话 ID: {result.session_id}")

    # 流式输出（逐 chunk 处理）
    async for chunk in client.run_stream("解释这段代码"):
        if chunk.type == "text_delta":
            print(chunk.content, end="", flush=True)
        elif chunk.type == "tool_call":
            print(f"\n[工具调用: {chunk.tool_name}]")

    # 工具过滤：只允许读操作
    result = await client.run(
        "分析代码质量",
        include_tools=["read_file", "grep", "glob"]
    )

事件系统

通过 Builder .on() 注册事件处理器，监听工具调用、错误等事件：

from cody import Cody, EventType

client = (
    Cody()
    .workdir(".")
    .on(EventType.TOOL_CALL, lambda chunk: print(f"→ {chunk.tool_name}"))
    .on(EventType.TOOL_RESULT, lambda chunk: print(f"← {chunk.tool_name} done"))
    .on(EventType.DONE, lambda chunk: print(f"完成，tokens={chunk.usage.total_tokens}"))
    .build()
)

自定义工具

通过 Builder .tool() 注册自定义异步工具函数，与内置工具享有相同的错误重试机制：

import httpx

async def fetch_jira_ticket(ticket_id: str) -> str:
    """从 Jira 获取工单详情"""
    async with httpx.AsyncClient() as c:
        resp = await c.get(f"https://your.jira.com/rest/api/2/issue/{ticket_id}")
        return resp.text

client = (
    Cody()
    .workdir(".")
    .tool(fetch_jira_ticket)
    .build()
)

CLI 参考

CLI 提供命令行方式调用 Cody，适合在终端中快速执行任务或集成到脚本中。

安装：pip install cody-ai[cli]

cody run

执行单次任务，完成后退出。

cody run [OPTIONS] PROMPT

选项：
  --model TEXT          指定模型（如 claude-sonnet-4-0）
  --workdir TEXT        工作目录（默认当前目录）
  --image PATH          附带图片（可重复使用，多模态）
  --session TEXT        续接已有会话
  --continue            续接最近一次会话
  --include-tools TEXT  逗号分隔的工具白名单
  --exclude-tools TEXT  逗号分隔的工具黑名单
  --max-tokens INT      Token 上限（熔断）
  --max-cost FLOAT      成本上限 USD（熔断）
  --max-steps INT       工具调用步数上限
  -v, --verbose         输出详细日志

示例：
  cody run "为 api.py 添加类型注解"
  cody run --image screenshot.png "修复这个 bug"
  cody run --include-tools grep,read_file "分析代码质量"
  cody run --max-tokens 50000 "重构整个模块"

cody chat

交互式多轮对话，支持流式输出。

cody chat [OPTIONS]

选项：
  --model TEXT      指定模型
  --workdir TEXT    工作目录
  --session TEXT    恢复已有会话

内置命令（对话中输入）：
  /exit   退出
  /clear  清空对话历史
  /save   保存当前会话

示例：
  cody chat
  cody chat --session ses_abc123  # 恢复对话

cody tui

全屏终端 UI（基于 Textual），适合长时间交互任务。安装：pip install cody-ai[tui]

cody tui [OPTIONS]

TUI 内置斜杠命令：
  /image <path> <msg>   发送图片给 AI（多模态）
  /skills               列出可用 Skills
  /skills enable X      启用指定 Skill
  /skills disable X     禁用指定 Skill
  /settings             查看当前设置
  /settings model X     切换模型
  /settings thinking on|off  切换思考模式

cody config

# 交互式配置向导（首次使用推荐）
cody config setup

# 查看当前配置（API Key 自动脱敏）
cody config show

# 设置单项配置
cody config set model claude-sonnet-4-0
cody config set enable_thinking true
cody config set thinking_budget 10000

cody skills

# 列出所有可用 Skills
cody skills list

# 查看 Skill 详情
cody skills show git

# 初始化项目（生成 CODY.md）
cody init

配置项说明

Cody 的配置存储在 ~/.cody/config.json（全局）和 .cody/config.json（项目级，优先级更高）。

环境变量

环境变量说明示例值
CODY_MODEL主模型名称claude-sonnet-4-0
CODY_MODEL_API_KEY主模型 API Keysk-ant-...
CODY_MODEL_BASE_URL主模型 API Base URL（OpenAI 兼容）https://api.openai.com/v1
CODY_SMALL_MODEL小模型（用于压缩/摘要，成本低）claude-haiku-3-5
CODY_CODING_PLAN_KEY阿里百炼 Coding Plan Keysk-sp-xxxxx
CODY_ENABLE_THINKING开启思考模式true
CODY_THINKING_BUDGET思考 Token 预算10000
CODY_SKILL_DIRS额外 Skill 搜索目录（逗号分隔）/my/skills,/team/skills
CODY_CORS_ORIGINSWeb 后端 CORS 白名单http://localhost:3000

安全配置

# .cody/config.json
{
  "security": {
    "allowed_roots": ["/data", "/shared"],
    "strict_read_boundary": true,
    "blocked_commands": ["rm -rf", "curl | sh"]
  }
}
  • allowed_roots：除 workdir 外还允许访问的目录列表（绝对路径）
  • strict_read_boundary：开启后，读操作（read_file、grep 等）也受边界限制
  • blocked_commands：自定义禁用命令模式（exec_command 检查）

多模型配置示例

# Claude（默认）
export CODY_MODEL_API_KEY='sk-ant-...'

# OpenAI
export CODY_MODEL='openai:gpt-4o'

# DeepSeek

# 智谱 GLM（OpenAI 兼容）

Agent Skills

Skills 是 AI 的专业知识模块，遵循 agentskills.io 开放标准，可跨 Claude Code、Cursor 等 26+ 平台复用。

Cody 内置 11 个 Skills：git、github、docker、npm、python、rust、go、java、web、cicd、testing。

创建自定义 Skill：

# 项目级 Skill（优先级最高）
mkdir -p .cody/skills
cat > .cody/skills/my-skill.md <<'EOF'
---
name: my-skill
description: 描述这个 Skill 的用途，AI 根据此决定何时加载
metadata:
  author: your-name
  version: "1.0"
---
# 我的自定义 Skill

AI 代理的操作指南...

## 示例
- 步骤一：...
- 步骤二：...
EOF

Skill 加载优先级：.cody/skills/（项目）> ~/.cody/skills/（用户）> 内置 Skills。

熔断器

熔断器防止失控 Agent 消耗过多资源，支持 Token 上限、成本上限和死循环检测：

from cody import Cody, CircuitBreakerConfig

client = (
    Cody()
    .workdir(".")
    .circuit_breaker(
        max_tokens=100_000,        # Token 上限
        max_cost_usd=2.0,           # 成本上限（USD）
        max_steps=50,               # 工具调用步数上限
        loop_detect_turns=5,        # 死循环检测窗口
        loop_similarity_threshold=0.85  # 相似度阈值
    )
    .build()
)

try:
    result = await client.run("复杂任务...")
except CircuitBreakerError as e:
    print(f"熔断触发: {e.reason}")  # token_limit / cost_limit / loop / step_limit

MCP 集成

通过 MCP（Model Context Protocol）连接外部服务器，扩展 Agent 能力（GitHub、数据库等）：

# stdio 传输（本地进程）
client = (
    Cody()
    .workdir(".")
    .mcp_stdio_server(
        name="github",
        command="npx",
        args=["@modelcontextprotocol/server-github"],
        env={"GITHUB_TOKEN": "ghp_..."}
    )
    .build()
)

# HTTP 传输（远程服务器，如飞书）
client = (
    Cody()
    .workdir(".")
    .mcp_http_server(
        name="lark",
        url="https://mcp.feishu.cn/sse"
    )
    .build()
)

子代理系统

Cody 支持孵化专业子代理（code / research / test / generic），asyncio 并发执行，最多 5 个并发：

# Agent 会自动使用 spawn_agent 工具
result = await client.run(
    """
    请并发执行以下任务：
    1. 分析 src/ 目录的代码质量（research agent）
    2. 运行所有测试并生成报告（test agent）
    3. 重构 utils.py 的错误处理（code agent）
    """
)

# SDK 直接管理子代理
agent_id = await client.spawn_agent(
    task="分析安全漏洞",
    agent_type="research"
)
status = await client.get_agent_status(agent_id)
子代理支持恢复（resume）：调用 client.resume_agent(agent_id) 可从上次中断处继续。
